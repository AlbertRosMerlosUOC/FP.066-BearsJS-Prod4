Creando la parte visible de nuestra aplicación. FrontEnd

En este producto hemos creado un PROYECTO PROPIO y en EQUIPO que ha seguido las indicaciones de un proyecto tipo Agenda Semanal usando las tecnologías base de desarrollo de frontal: HTML, CSS y JS, y que el equipo HA ADAPTADO LAS INDICACIONES AL PROYECTO PRESENTADO. Para ello hemos incorporado tanto el framework Bootstrap como las API de HTML que nos permitan dar la interactividad a nuestro diseño.

En este producto hemos dejado listas las opciones básicas de nuestro frontal, en el caso del proyecto: semanas del mes, días de esa semana, creación de tareas a hacer y la posibilidad de mover tareas entre los días de la semana, para ello necesitamos que nuestra aplicación sea sencilla y fácil de usar por parte de los usuarios, y para ello las herramientas de frontal nos ayudarán a ese desarrollo.

Sirviendo datos con GraphQL y MongoDB

En este producto hemos implementado un servidor con Node/Express en el que serviremos la información a partir del gestor de bases de datos MongoDB y utilizando la especificación GraphQL para proveer de los argumentos para realizar las Queries y los Mutations para interactuar con el frontal en el siguiente producto.

Añadiendo, modificando y borrando información

En este producto modificaremos el producto 2 para poder interactuar de forma completa con los datos de la base de datos añadiendo las mutaciones correspondientes a nuestro API, así como las modificaciones necesarias en nuestro frontal para poder interactuar.

Trabajando en tiempo real de forma colaborativa

Finalizamos nuestro proyecto implementando los mecanismos para el trabajo colaborativo mediante la comunicación en tiempo real de datos por la subscripción a estos con la incorporación de un sistema de Pull/Push a través de websockets y PubSub.
